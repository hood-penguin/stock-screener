## 컴포넌트별 책임

### 클라이언트 레이어
- **Web (Next.js)**: SSR로 초기 로딩 성능 확보, SEO 대응
- **Mobile (React Native)**: iOS/Android 동시 지원, 푸시 알림

### API Gateway
- JWT 토큰 검증
- 분당 요청 수 제한 (일반 100req/min, 프리미엄 1000req/min)
- 요청 로깅 및 모니터링

### API Server (FastAPI)
- 비동기 처리 (`async/await`)
- 스크리닝 결과 조회 및 필터링
- 사용자 맞춤 기준 설정 저장

### Screener Engine (Celery Workers)
- **핵심**: 플러그인 기반 스크리닝 기준 평가
- 매일 장 마감 후 자동 실행
- 결과를 DB에 저장

### Data Collectors
- 각 시장별 어댑터 패턴으로 구현
- 원시 데이터 보존 → 정제 → 저장
- 실패 시 재시도 로직

## 데이터 흐름

```
[외부 API] → [수집기] → [정제/정규화] → [원시 테이블]
                                              ↓
                                    [스크리닝 엔진 실행]
                                              ↓
                                    [점수/결과 테이블]
                                              ↓
                              [API Server가 캐시 후 서빙]
                                              ↓
                                    [Web/App 클라이언트]
```

## 확장 포인트

| 확장 항목 | 방법 |
|---|---|
| 새 스크리닝 기준 추가 | `criteria/` 디렉터리에 플러그인 파일 추가 |
| 새 시장 추가 (예: 일본) | MarketAdapter 인터페이스 구현 |
| 새 데이터 소스 추가 | DataProvider 인터페이스 구현 |
| 새 알림 채널 추가 | NotificationChannel 인터페이스 구현 |
